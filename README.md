# My C++ Progress

This is my personal space where I keep all the code I write. I am using this to track my journey from the absolute basics until I (hopefully) reach a professional level. It is a way for me to archive my logic exercises, university assignments, and the roadmap I am following.

### How it's organized
I have sorted the folders based on the source of the work:
- **Roadmap Levels:** These are the problem-solving and logic exercises from the learning path I'm taking.
- **University:** This includes my homework, lab problems, and projects from my university studies.
- **Shared Code:** These are small libraries and headers I wrote so I don't have to repeat the same code in every project.

### A Quick Note
To keep this space clean, I don't upload `.exe` files or IDE folders. Only the source code is here. If you want to run anything, you will need to compile it yourself:
`g++ main.cpp -o main`
`./main`

### Why am I doing this?
I want to look back at this in a few years and see the progress I've made. My early code might be simple or even a bit messy, but that's just how the learning process works.

# مساحتي في برمجة ++C

هذا المكان هو بمثابة أرشيف شخصي لكل كود أكتبه خلال رحلتي في البرمجة. هدفي من هذا العمل هو توثيق تطوري من البدايات وحتى الوصول إلى مستوى الاحتراف بإذن الله، حيث أضع هنا تمارين المنطق، تكليفات الجامعة، والدروس التي أتعلمها ضمن خطة (Roadmap) محددة.

### كيف قمت بتنظيم العمل؟
قسمت الملفات حسب مصدرها الأساسي:
- **مستويات الخطة (Roadmap):** تمارين حل المشكلات والمنطق البرمجي التي أعمل عليها لتطوير مهاراتي.
- **الدراسة الجامعية:** تشمل الوظائف، المسائل البرمجية، والمشاريع الخاصة بمواد البرمجة في جامعتي.
- **أكواد مشتركة:** ملفات "هيدر" ودوال كتبتها مرة واحدة لأتمكن من استخدامها في عدة مشاريع دون الحاجة لتكرار الكود.

### ملاحظة تقنية
أحرص على إبقاء هذه المساحة نظيفة، لذلك لا أرفع ملفات التشغيل (exe) أو مجلدات الإعدادات الخاصة ببيئة التطوير. المرفوع هنا هو الكود المصدري فقط. إذا أردت تشغيل أي برنامج، ستحتاج لعمل (Compile) يدوياً:
`g++ main.cpp -o main`
`./main`

### لماذا أقوم بهذا؟
أريد أن أعود إلى هنا بعد عدة سنوات لأرى كيف بدأت وكيف تطورت. قد تكون الأكواد الأولى بسيطة جداً أو غير منظمة تماماً، لكنني أعتبر ذلك جزءاً أساسياً من رحلة التعلم.
